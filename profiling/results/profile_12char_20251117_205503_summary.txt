## Root Cause Analysis

The 95% memory throughput with 12-char words indicates:
1. **Memory writes are STILL the bottleneck** (not reads - shared memory fixed that)
2. **Longer words = more bytes to write** (13 bytes vs 5 bytes per word)
3. **Write coalescing may be poor** - threads writing to non-contiguous addresses

## Next Optimization Priorities

### Priority 1: Improve Write Coalescing
**Problem**: Each thread writes (word_length + 1) bytes to separate locations
- 12-char: tid * 13 bytes = potentially unaligned, non-coalesced writes

**Solutions**:
a) Pad output to 16-byte alignment (13 ‚Üí 16 bytes per word)
b) Use shared memory staging + cooperative writes
c) Restructure output buffer for better coalescing

**Expected gain**: 2-5x (could reduce memory throughput from 95% to 20-40%)

### Priority 2: Barrett Reduction
Now lower priority since compute is only 18% utilized.
Only worth pursuing after fixing memory bottleneck.

### Priority 3: Increase Batch Size
The profiler suggests the workload may be too small.
Test with 500M-1B word batches to saturate the GPU better.
